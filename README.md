## 🧪 과제 1: 데이터 모델링

### 학습 내용
- ERwin을 활용한 데이터베이스 모델링
- 엔터티(Entity), 속성(Attribute), 관계(Relationship) 정의
- 기본키(PK) 및 외래키(FK) 설정
- 개념 / 논리 / 물리 모델 설계

### 학습 포인트
- 요구사항을 데이터 구조로 변환하는 과정 이해
- 데이터 무결성과 관계 설정의 중요성
- 정규화 관점에서의 테이블 설계 능력 향상

---

## 🧪 과제 2: 데이터베이스 설계 및 구현

### 학습 내용
- 데이터 모델을 기반으로 한 데이터베이스 설계
- 실제 데이터를 처리하는 프로그램 구현
- 데이터 접근 및 처리 흐름 분석
- 설계 결과를 문서로 정리하여 체계적으로 설명

### 학습 포인트
- 설계된 데이터 모델이 실제 구현으로 연결되는 과정 이해
- 데이터 구조가 프로그램 로직에 미치는 영향 분석
- 데이터베이스 시스템의 실질적 활용 관점 학습

---

## 📘 학습 정리

- 데이터베이스 설계는 단순한 테이블 생성이 아닌  
  **요구사항 분석과 구조적 사고의 결과**임을 이해
- ERD 설계부터 구현까지의 흐름을 경험
- 데이터 무결성과 확장성을 고려한 설계의 중요성 인식
